6. DEFECT FOUND & RESOLVED (during the run)
--------------------------------------------------------------------------------
Defect          : The first-issued bundle's test feeder was a Python script
                  requiring 'paho'. The customer's SYSTEM python3 has no paho,
                  and Debian/Ubuntu (PEP-668, externally-managed) blocks
                  'pip install'. Feeder failed: ModuleNotFoundError: paho.
                  Secondary friction: feeder default port (1883) did not match
                  the auto-bumped local port (1884).

Resolution      : Replaced the Python feeder with sensor_feed.sh using
                  mosquitto_pub (mosquitto-clients is already installed by
                  install.sh — zero extra dependency). It auto-detects the local
                  port from gateway.conf, so there is no port to remember.
                  install.sh now prints a ready-to-run 'bash sensor_feed.sh'
                  hint with the resolved port.

Verification    : New shell feeder validated in a Debian container (all 5 topics
                  published, awk math correct). Re-install on WSL was clean and
                  'bash sensor_feed.sh' worked first try.

Commit          : cf558a5 "fix: ship mosquitto_pub shell test feeder
                  (remove paho/PEP-668 friction)"
