4.2 Bridge Script (mqtt_to_pubsub.py) � Rewrite
The rewritten bridge handles multi-site topics, validates schema, updates Redis live state, and runs as a systemd service. Key changes from POC version: topic pattern microgrid/+/telemetry, schema validation against expected 13 fields, Redis HSET per site_id, publishes to microgrid-telemetry Pub/Sub.

# Key structural changes vs POC script
TOPIC_PATTERN = 'microgrid/+/telemetry'          # was: v1/devices/victron/telemetry
PUBSUB_TOPIC  = 'microgrid-telemetry'            # was: victron-telemetry
REDIS_HOST    = 'localhost'                       # new: live fleet state cache
REQUIRED_FIELDS = [                              # new: schema validation
    'site_id','timestamp','battery_v','battery_soc','battery_current',
    'battery_temp','ac_input_v','ac_output_v','ac_output_i',
    'solar_w','load_w','inverter_state','power_balance_w'
]

def on_message(client, userdata, msg):
    payload = json.loads(msg.payload)
    site_id = msg.topic.split('/')[1]            # extract from topic
    payload['site_id'] = site_id
    payload['timestamp'] = payload.get('timestamp', datetime.utcnow().isoformat())
    # Validate schema
    missing = [f for f in REQUIRED_FIELDS if f not in payload]
    if missing: log.warning(f'Missing fields {missing} from {site_id}')
    # Update Redis live state (TTL 120s)
    redis_client.hset(f'site:{site_id}', mapping=payload)
    redis_client.expire(f'site:{site_id}', 120)
    # Publish to Pub/Sub
    pubsub_client.publish(pubsub_topic_path, json.dumps(payload).encode())

# systemd service: /etc/systemd/system/mqtt-bridge.service
[Unit]
Description=MQTT to Pub/Sub Bridge
After=network.target mosquitto.service
[Service]
ExecStart=/usr/bin/python3 /opt/microgrid/mqtt_to_pubsub.py
Restart=always
RestartSec=10
User=microgrid
[Install]
WantedBy=multi-user.target

5. ML Anomaly Detection Layer
5.1 Feature Engineering (BigQuery SQL View)
A BigQuery view computes the derived features that the Isolation Forest model consumes. These are not stored raw but computed at query time to keep the table schema clean.

CREATE OR REPLACE VIEW microgrid_db.ml_features AS
SELECT
    site_id, timestamp,
    battery_v, battery_soc, battery_current, battery_temp,
    ac_input_v, ac_output_v, ac_input_power, ac_output_power,
    solar_w, load_w, inverter_temp, power_balance_w,
    -- Derived features
    battery_v / 24.0 AS battery_v_norm,
    ac_output_power / NULLIF(ac_input_power + solar_w, 0) AS efficiency_ratio,
    LAG(battery_soc) OVER (PARTITION BY site_id ORDER BY timestamp) AS prev_soc,
    (battery_soc - LAG(battery_soc) OVER (PARTITION BY site_id ORDER BY timestamp))
        / TIMESTAMP_DIFF(timestamp,
          LAG(timestamp) OVER (PARTITION BY site_id ORDER BY timestamp), SECOND)
        * 60 AS soc_rate_per_min,
    COUNT(*) OVER (PARTITION BY site_id,
        TIMESTAMP_TRUNC(timestamp, HOUR)) AS mode_transitions_per_hour
FROM microgrid_db.microgrid_telemetry
WHERE timestamp > TIMESTAMP_SUB(CURRENT_TIMESTAMP(), INTERVAL 7 DAY)

6. Multi-Agent Layer (LangGraph)
6.1 Framework Decision
LangGraph is selected over CrewAI and raw LangChain for the following reasons specific to this project:
� Explicit state machine model maps to ISA-101 supervisory control hierarchy and PhD safety supervisor framework
� LangGraph interrupt() node provides the human-in-the-loop approval gate natively
� LangSmith tracing gives auditable decision logs � critical for safety narrative at conferences
� Most recognised at ODSC East and AI Dev World audiences � reduces explanation overhead
� Claude Sonnet (claude-sonnet-4-20250514) via Anthropic API as LLM backbone � best reasoning quality for industrial context
